Simple way to Run Rules Engine. 
There are multiple modules present in this project.

- praecepta-rules-core represents how to create Criteria [Multi Condition with Join Operators] and Actions
- praecepta-rules-evaluators represents how to Evaluate the Citerias provided
- praecepta-rules-actions represents how to trigger the Actions depending on Either Citeria Matched and UnMatched
- praecepta-rules-dao represents how to Persist Criterias, Actions information to Different Persistant Layers [SQL DB, NoSQL DB, Files etc]
- praecepta-rules-setup represents the provision to Pull Criterias, Actions information ffrom Different Persistant Layers [SQL DB, NoSQL DB, Files etc] and Keep it in JVM memory
-  praecepta-rules-engine to run end to end engine
